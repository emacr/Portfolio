    decision: "Apache Flink was chosen for its stream-first programming model (not micro-batches), native exactly-once semantics support, and better performance on stateful operations with RocksDB backend."
    consequences: "Lower latency (true streaming vs. Spark's 1s micro-batches). Steeper learning curve for the team. 2 weeks of enablement were invested before production."
  - title: "ADR-002: Self-hosted MinIO as S3-compatible Data Lake"

    decision: "Self-hosted MinIO on bare-metal was chosen for cost control and data sovereignty. The S3-compatible API allowed using the same tooling ecosystem (dbt, Metabase) without changes."
    consequences: "~70% reduction in storage costs vs. managed S3. Additional operational responsibility for backup, mitigated with restic + automatic rotation."

## Project Context

The need arose from a retail client making business decisions with day-old data. This pipeline eliminated that lag, enabling real-time price and stock adjustments based on current purchase behavior.

## Business Impact

Real-time analytics reduced the response time to inventory anomalies from **4 hours to 45 seconds**, improving conversion by 12% in the first 30 days.
